Use case : curs online profesor - studenti

Affordances

  • un utilizator este capabil sa comunice cu alt ultilizator sau cu un grup de utilizatori folosind mesaje text sau prin apeluri video/audio
  • un utilizator conectat la un canal video/audio poate scrie pe alte canale in timpul sesiunii
  • detinatorul serverului poate crea canale la care doar anumiti utilizatori au acces sau au doar drept de citire etc.
  • detinatorul serverului poate atribui roluri utilizatorilor intrucat sa aiba diferite drepturi pe server precum: accesul la diferite canale, permisiuni de a modifica drepturile altor utilizatori
  • un utilizator poate partaja ecranul sau anumite ferestre in cadrul unui canal
  • un utilizator poate incarca fisiere de pe dispozitivul sau pe un anumit canal
  • in timpul unei sesiuni utilizatorul poate separa fereastra continutului partajat de fereastra principala a aplicatiei

Visual Variables

Structura

  • Organizează mesajele în canale, astfel încât utilizatorii să poată vizualiza și accesa mesajele într-un mod structurat. Acest lucru poate include etichete sau categorii pentru canale și filtre de căutare.
  • In partea stanga a ferestrei se afla iconitele serverelor, lista cu canalele,utilizatorii conectati la un canal(care apar sub numele canalului), canalul pe care este conectat utilizatorul cu functiile pe care le poate folosi, respectiv numele utilizatorului si imaginea de profil cu bulina verde sau rosie care reprezinta statusul, restul spatiului fiind ocupat de chat sau in cazul unui canal de voce/video in acest spatiu se vor regasi imaginile utilizatorilor sau partajarea ecranului.

Design

  • Design-ul interfeței de utilizator: are un design modern și atrăgător, cu elemente de interfață clare și ușor de utilizat.

Marime

  • Marimea elementelor difera: Iconitele pentru navigare si descoperirea caracteristicilor mici deoarece navigarea este simpla si intuitiva
  • Elementele mari sunt lista cu canale si imaginile respectiv camerele video ale participantilor sau ecranul partajat.

Culori

  • Culorile principale folosite sunt neutre: negru, gri si alb. Imaginile si iconitele pot avea culori diferite, de asemenea textul poate fi editat cu ajutorul unor taguri sa arate intr-un mod diferit fata de cel default
  • In timpul sesiunii un utilizator se poate deconecta prin apasarea unui buton de culoare rosie

Interaction flow

  1. Un utilizator creeaza un server si poate distribui un link de invitatie altor utilizatori
  2. Alti utilizator acceseaza acel link si se conecteaza la acel server si vor avea acces la canalele existente pe acel server
  3. Utilizatorii se vor conecta la un anumit canal si vor avea acces la continutul partajat si convorbirea purtata in cadrul acelui canal
  4. Utilizatorii se pot deconecta in orice moment prin apasarea butonului rosu "deconecteaza-te" sau prin apasarea butonului pentru deconectare din partea stanga a ferestrei care apare in momentul conectarii la canal

Metaphors and Idioms

  1. Metafore
    • Iconitele pentru audio (microfon), pentru sunet(castile) si pentru video (camera), primele doua apar atat in versiune normala, adica in modul "Pornit", cat si in modul "Oprit" acestea fiind taiate de o linie rosie;
    • Iconite pentru a folosi un emoticon;
    • butonul Exit, reprezentat de un microfon langa care se afla un "x", intr-o bulina rosie;
    • Iconita pentru partajarea ecranului(ecran cu o sageata);
  2. Idiome
    • Butonul pentru a incepe o activitate reprezentata de o iconita "racheta"
    • Butonul pentru a scoate playeru in alta fereastra, reprezentat de iconita patrat cu o sageata inspre exterior

Aplicatia discord are implementata sectiunea de text only, adica utilizatorii nu trebuie sa fie conectati pe un canal audio ca sa scrie.

Pentru interactiune strict fara componente audio:
  • In partea stanga sub un anumit canal sa apara numele utilizatorilor care scriu pe acel canal
  • In dreptul utilizatorului sa fie o iconita "tastatura" alba, daca a trimis un mesaj in ultimele 5 minute, rosie in caz contrar

Cognitive overload

Aplicatia discord permite utilizatorilor să creeze mai multe canale pentru diferite subiecte sau scopuri. În timp ce acest lucru poate fi util pentru organizare, poate fi și copleșitor pentru studenți care încearcă să țină pasul cu mai multe conversații care se desfasoare in mai multe canale în același timp.

Acest lucru poate fi particular dificil pentru studenții care nu sunt familiarizați cu aplicatia sau care au dificultăți în multitasking. Aplicatia permite utilizatorilor să împărtășească o varietate de media bogată, inclusiv imagini, videoclipuri și link-uri. În timp ce acestea pot fi utile pentru învățare, pot fi și copleșitoare dacă prea multă informație este prezentată odată. Acest lucru poate fi exacerbate dacă elevii încearcă să țină pasul și cu conversațiile de tip text în același timp.

Improvements

  1. Problema: Dificultate în gestionarea serverelor mari cu mai multe canale și utilizatori.

    • Soluție: Introducerea unor instrumente de moderare mai avansate pentru proprietarii și administratorii serverului, cum ar fi posibilitatea de a șterge mesaje sau utilizatori în masă, a seta roluri și permisiuni personalizate și a programa mesaje. Permiterea utilizatorilor să creeze subcanale în cadrul canalelor pentru a organiza conversațiile mai eficient.

  2. Problema: Design inconsistent între aplicațiile desktop și mobile.

    • Soluție: Asigurarea consistentei în design-ul general, aspectul și interfața utilizatorului pe ambele platforme, astfel încât utilizatorii să poată trece ușor între cele două fără confuzie. Utilizarea unor elemente de design responsive optimizate pentru ecranele de desktop și mobile pentru a oferi o experiență fără probleme.

  3. Problema: Integrare limitată cu alte aplicații și servicii.

    • Soluție: Permiterea utilizatorilor să conecteze aplicatia Discord la alte aplicații și servicii, cum ar fi Google Drive, Trello sau GitHub, pentru a face mai ușor colaborarea la proiecte și partajarea de conținut. Oferirea mai multor opțiuni de personalizare pentru integrări, cum ar fi posibilitatea de a alege ce canale sau utilizatori să fie notificați pentru evenimente specifice.

Fitt's, Hick's, Steering

Legile empirice ale lui Fitts, Hick și Steering sunt utilizate în mod obișnuit în contextul interacțiunii om-calculator pentru a descrie modul în care oamenii interacționează cu dispozitivele de intrare precum mouse-urile, tastaturile și touchscreen-urile. Când luați în considerare interacțiunea multi-device pentru Discord, aceste legi pot fi în continuare aplicabile, dar acestea trebuie adaptate:

Legea lui Fitts

Legea lui Fitts descrie relația dintre dimensiunea unei ținte și timpul necesar pentru a ajunge la ea cu un dispozitiv de indicare.

În contextul interacțiunii multi-device pentru Discord, această lege poate fi încă relevantă, dar țintele pot fi răspândite pe mai multe dispozitive, cum ar fi un computer desktop și un telefon mobil. Ca rezultat, este posibil ca dimensiunea țintelor și distanța dintre ele să fie ajustate pentru a ține cont de diferitele dimensiuni ale ecranului și metode de input.

Legea lui Hick

Legea lui Hick descrie relația dintre numărul de opțiuni și timpul necesar pentru a lua o decizie.

În contextul aplicatiei Discord, utilizatorii pot avea mai multe opțiuni pentru a interacționa cu platforma pe mai multe dispozitive, cum ar fi chat vocal, chat text și chat video. Pentru a reduce timpul de decizie, s-ar putea lua în considerare gruparea funcțiilor sau opțiunilor similare între dispozitive pentru a minimiza numărul de opțiuni prezentate utilizatorului la un moment dat.

Legea lui Steering

Legea lui Steering descrie relația dintre mișcarea unui cursor și direcția mișcării.

În contextul interacțiunii multi-device pentru Discord, este posibil ca cursorul să nu fie întotdeauna prezent, deoarece utilizatorii pot interacționa cu platforma prin ecrane tactile sau comenzi vocale.Deci aplicatia are nevoie de metode alternative de input, cum ar fi gesturile sau recunoașterea vocii, pentru a permite utilizatorilor să interacționeze cu platforma într-un mod natural și intuitiv pe mai multe dispozitive.

Features

  1. Magazin de jocuri: ar putea integra o funcție de magazin de jocuri care permite utilizatorilor să cumpere jocuri și alte articole legate de jocuri direct prin platformă.

    Utilizatorii ar putea accesa magazinul printr-o filă separată de interfața aplicatiei și sa navigheze în jocuri după gen, platformă și popularitate. Când un utilizator cumpără un joc, acesta ar putea fi adăugat automat în biblioteca sa pe Discord și ar putea vedea ce jocuri joacă prietenii lor.

  2. Instrumentul de transcriere: s-ar putea adăuga un instrument de transcriere care transcrie automat conversațiile prin chat vocal și video în text.

    Acest lucru ar face mai ușor pentru utilizatori să citească mesajele si să găsească informații importante fără a fi nevoiți să asculte conversații întregi. Instrumentul de transcriere ar putea fi accesat printr-un buton din fereastra de chat, iar utilizatorii ale să îl pornească sau să îl dezactiveze.

  3. Calendar: s-ar putea adăuga o funcție de calendar care permite utilizatorilor să programeze evenimente și mementouri direct prin platformă.

    Utilizatorii pot crea și gestiona evenimente dintr-o filă separată de interfata. Calendarul se poate sincroniza cu alte aplicații de calendar, cum ar fi Google Calendar, pentru a se asigura că utilizatorii nu pierd evenimente importante.

Iată trei îmbunătățiri de accesibilitate care ar putea fi aduse Discord, bazându-se pe funcțiile de accesibilitate existente:

  1. Suport îmbunătățit pentru cititorul de ecran: în prezent discordul ofera un anumit suport pentru cititorul de ecran, dar ar putea fi îmbunătățit prin adăugarea de etichete mai descriptive pentru butoane și câmpuri, asigurându-se că comenzile rapide de la tastatură sunt accesibile prin cititoarele de ecran și îmbunătățind experiența generală de navigare pentru utilizatorii care se bazează pe acestea.
  2. Mod contrast: pe discord exista o temă deschisă și întunecată, dar s-ar putea adauga și un mod de contrast. Modul de contrast ar putea fi util în special pentru utilizatorii cu deficiențe de vedere și ar putea îmbunătăți lizibilitatea și gradul de utilizare general al platformei.
  3. Subtitrări îmbunătățite: subtitrările pentru chat-urile vocale și video ar putea fi îmbunătățite prin adăugarea de câteva funcții, cum ar fi capacitatea de a personaliza dimensiunea și culoarea fontului, capacitatea de a ajusta poziția subtitrărilor pe ecran, și capacitatea de a activa și dezactiva rapid subtitrările în timpul unui chat.

Webex

Use case: sedinta online student - profesor indrumator de licenta

Affordances

A user who uses this communication application is able to:

  • communicate with another user or group of users, either using an audio system or text messages;
  • use the device's camera to create a new interaction interface;
  • share the screen live or individually various windows;
  • share files that are on the device used;

Visual Variables

The visual aspects of the elements that make up the communication interface, position, size, color, etc. helps users to focus on communicating with the rest of the people participating in the conversation.

Position

  • the element/image that represents the main speaker is always placed in the center of the interface, replacing, if necessary, the previous speaker.
  • if they are activated, the chat for text messages and the list of participants are positioned on the right side, the chat being placed at the bottom to be more easily accessible.
  • bthe application-specific buttons are placed separately from the previously mentioned elements, also at the bottom for accessibility.

Size

  • the largest element of the interface is the element of the main speaker, possibly replaced by his video camera, this element will prioritize (if applicable) to show the screen shared by a participant;
  • the elements for the list of participants and chat are smaller because they offer a moderate level of information;
  • the functional buttons are the smallest elements in the interface, being rarely needed, some being used only once within a single conversation;

Colour

  • the application only uses neutral colors, black, shades of gray, white writing, the only colors used being green for connecting and red for closing the call. This color scheme does not contain gaudy shades so as not to distract users.

Interaction flow

For the conversation itself, we can outline the following flow:

  • a user starts a communication room being assigned host;
  • other users who intend to participate, use a special link to enter the respective room;
  • opening the link presents users with an interface that confirms information about the user, and that allows them to set certain options (camera, microphone, both the component used and whether it is turned on or not) before participating in the conversation;
  • once entered the room, the user can participate in the conversation and has access to all the functionalities presented in the "Affordances" category.
  • once the purpose of the conversation is completed, the user can disconnect using the "Leave meeting" button

Metaphors and Idioms:

The metaphors used to easily illustrate the functionalities to the user are:

  • The icons for audio (microphone) and video (camera), both in the normal version, white outline and in the "Off" mode using the red color instead;
  • Icons for accessing the list of participants, chat;
  • IIcons to raise your hand or use another emoticon;
  • the Exit button, using an X;
  • File sharing

For a text-only type of interaction, we can implement several columns in the following way:

  1. Participants column:
    • Contains the list of all participants, those who participate verbally in the conversation will have a white dot to the left of their name;/li>
  2. Chat column;
    • Contains the messages sent, in order, accompanied by who sent the message and when.
  3. Options column:
    • Show/hide participants/chat;
    • Share file;
    • Leave the call
    • etc

Idioms, icons that do not accurately describe the purpose of the button, introducing new concepts to the user:

  • The button for sharing, accompanied by the icon for New Whiteboard

Cognitive overload

There is a possibility of cognitive overload in the following case: Several users are connected to the communication room, each with the microphone on (and the camera). All users want to communicate something, too many discussions are heard, which becomes worse if these discussions are also on different topics. In this chaotic scenario, there is too much information for any user, making it difficult to process the information.

Improvements

Webex being mainly an application for serious conversations, both the comfort of the users and the integrity of the conversation must be ensured, thus, we propose the following improvements, these improvements do not depend on the platform used and can be implemented in a general way, either native mobile, native desktop or Web :

  1. Maximum volume tolerated:

    This improvement would allow users with sensitive hearing to set a maximum level for the volume that could stop the audio that exceeds this level, or even modify it to be reduced to the maximum level allowed.

  2. Maximum speakers tolerated:

    This improvement aims to stop situations similar to the scenario presented in the "Cognitive overload" chapter, each audio input will go into a slot that is allowed to communicate verbally with the users. If all slots are occupied, the maximum is reached, and any audio input is ignored until slots are freed. The number of slots can be preset before the creation of the communication room, or constantly modified by the host;

  3. Auto-off/censor shared content:

    This improvement allows the user to start a system that either censors or completely stops a potential video or audio input that is not beneficial to the conversation. There will be options to select what kind of information is detected by this system: violence, vulgar language or even themes that aim to distract attention from the main conversation.

Fitt's, Hick's, Steering

  1. Within this empirical law, we focus on the time required to move the cursor between the buttons:
    • The buttons are generally located either on the right side of the screen, for functionalities that do not influence the conversation, or at the bottom for those that have this possibility. Thus we have buttons grouped by functionality, at a relatively short distance;
    • The size of the buttons is kept small so as not to distract users, however, it is a comfortable size to allow easy operation by users;
  2. Hicks' law mentions the relationship between the number of options at a given moment, and the time to make a choice:
    • Being already limited in number of options, we can give as an example the sharing scenario: The user is presented with several menus grouped by the type of sharing: Screen, File, Camera. Thus, not all options are always presented and the user can quickly choose the desired sharing method.
  3. The time required to travel through a 2-dimensional tunnel is relatively small, which is supported by the other two empirical laws, the time to navigate through the interface and the time to choose one of several options being kept at small intervals.

Features

Request file

This function can be activated by any user to make file transfer easy. When starting this setting, the person requesting the files can include both a description of the desired file and a condition for the date of the last significant change in the requested file. (Ex: a teacher asks some students for homework)

Remote helper

This function allows a user1 to ask another user2 to share the screen of only user1 and create a text window and a cursor instance on the screen of user2, practically both users work on the same screen. The 2 elements are operated by user1 and can be used to guide user2. We can imagine a functionality similar to the TeamViewer application, BUT there is no remote connection to another computer, only elements that make remote guidance easier are instantiated.

App hosted share

This functionality allows a user to upload a document in any format to be viewed without sharing the document itself or the screen, thus the quick and temporary information will not take up space on the device of other users and there is no risk that the information will not be visible from due to a bad connection during screen sharing.
